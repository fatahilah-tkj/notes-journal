📘 Dokumentasi Pembuatan Website Field Notes

Dibuat: 12 Juni 2026 | Versi: 2.1 (Final dengan Paginasi)

1. Latar Belakang

Website ini dibangun sebagai jurnal belajar IT dan showcase proyek web. Berbasis Vanilla HTML, CSS, JavaScript tanpa framework, dioptimasi untuk pengelolaan dari HP serta deployment di Vercel.

💡 Kolaborasi Manusia + AI

Seluruh kode dalam proyek ini ditulis dengan bantuan Large Language Model (AI) berdasarkan Product Requirement Document (PRD) yang saya susun sebagai Lead Developer & Content Architect. Saya bertanggung jawab penuh atas arsitektur sistem, struktur data, desain visual, proses debugging, deployment, serta integrasi build script ke Vercel – semuanya dikelola dari perangkat seluler. Peran AI adalah membantu menerjemahkan spesifikasi teknis menjadi kode HTML/CSS/JS yang efisien dan sesuai standar.

3. Fitur Utama (Sesuai PRD Final v2.1)

  • Efek kertas bergaris – latar krem, garis horizontal biru, garis tepi merah (skeuomorphic binder).
  • Tab pisah – “Semua Catatan” dan “Daftar Latihan Web”, masing-masing dengan sorting & paginasi independen.
  • Pencarian real-time + tombol ✕ (muncul saat input diisi).
  • Paginasi 20 item per halaman – nyaman untuk ratusan data.
  • Routing dinamis – ?page=slug menampilkan isi catatan tanpa reload, tombol “kembali ke Daftar” mereset semua state.
  • Build otomatis – saat deploy, build.js membaca folder /catatan dan /proyek, lalu menghasilkan file daftar-catatan.json dan daftar-proyek.json dalam format minified.
  • Mobile-first & zero framework – semua kode vanilla, diketik dan diedit langsung dari HP (Acode/Spck).

4. Alur Kerja Pengelolaan Konten

  1. Tambah catatan baru → buat file .html di folder /catatan (isi parsial tanpa <html>).
  2. Tambah proyek baru → buat file .json di folder /proyek dengan properti: judul, deskripsi, link, tanggal.
  3. Jalankan build → node build.js (atau push ke GitHub → Vercel menjalankannya otomatis).
  4. Deploy → setiap git push memicu redeploy di Vercel.

5. Teknis Build Script (build.js)

Membaca semua file .html di /catatan, mengambil mtime sebagai tanggal, membuat slug dan judul otomatis. Juga membaca semua file .json di /proyek. Output ditulis ke root dengan JSON.stringify(data) (minified).

6. Frontend Logic (script.js)

  • fetch() kedua JSON → menyimpan ke variabel global.
  • Render daftar berdasarkan tab aktif, filter (search), sorting, dan halaman.
  • Event listeners untuk tab, sorting, search, tombol clear, paginasi, dan routing.
  • Saat mode baca (?page=...) → fetch file HTML parsial, tampilkan + tombol kembali. Kembali ke indeks mereset semua state (search kosong, tab catatan, sorting terbaru, halaman 1).

7. Desain Visual (CSS)

Menggunakan :root untuk variabel warna dan font. Efek kertas diterapkan di .paper-container dengan background-image: linear-gradient(...) dan ::before untuk garis merah. Semua komponen (tab, search, paginasi, tombol) dibuat responsif dengan media queries.

8. Deployment ke Vercel

Setelah repository dihubungkan ke Vercel, atur:

  • Build Command: npm run build
  • Output Directory: .

Vercel akan menjalankan build.js dan mendeploy file statis. Otomatis memperbarui setiap push ke branch utama.

9. Hal yang Dipelajari & Kendala

  • Mengatur efek garis kertas tanpa gambar background agar loading cepat.
  • Membangun routing dinamis murni dengan Vanilla JS dan URLSearchParams.
  • Mengimplementasikan paginasi + state per tab tanpa library.
  • Memastikan tombol clear search muncul dan berfungsi dengan event input.
  • Menangani graceful fallback saat fetch JSON atau HTML parsial gagal.
  • Menyusun build script Node.js yang membaca file system dan menghasilkan JSON minified.
  • Kolaborasi dengan AI: menyusun PRD, memerintahkan eksekusi kode, debugging, dan integrasi sistem.
